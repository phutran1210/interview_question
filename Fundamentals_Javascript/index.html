Variables - Biến

Biến được sử dụng rất nhiều trong bất kỳ một ngôn ngữ nào đó.
Nhiệm vụ của nó là là chứa đựng dữ liệu.
Biến có thể sử dụng để chứa rất nhiều dữ liệu khác nhau.

Tổng quan

Để sử dụng biến thì chúng ta cần phải khai báo nó trước, để JS có thể biết được.
Hiện giờ chúng ta có 3 cách để khai báo biến

							
							var a; // khai báo 1 biến có tên là a

							// let và const là 2 cách khai báo mới được thêm vào tại ES6
							let b;
							const c = 3;

Sau khi khai báo biến thì có thể gán cho nó một giá trị, bất kỳ giá trị nào mà bạn muốn.
Chúng ta cũng có thể khai báo rồi gán ngay lúc khai báo 

							
							var a; // khai báo 1 biến có tên là a

							a = 'Hello Javascript'; // gán a 

							alert(a); // gọi biến thông qua tên
var a; a = 'Hello Javascript'; alert(a);


Chúng ta có thể khai báo nhiều biến cùng một lúc 

							
							var a = 3, b = 5, c = 9;

							// Hoặc là viết nhiều dòng
							var a = 3,
								b = 5,
								c = 9;


							// hoặc là viết nhiều lần
							// cách này lâu nhưng mà nó dễ đọc nhất
							var a = 3;
							var b = 5;
							var c = 9;


Giải Thích

Hãy xem variable như là một cái hộp mà bạn có thể bỏ bất kỳ thứ gì vào
Cái hộp đó có một cái nhãn ở trên đó, cái này cũng là tên của variable
Sau khi gán giá trị rồi thì vẫn có thể gán lại một lần nữa
Khi gán 1 giá trị khác thì giá trị cũ sẽ bị xóa đi
Chúng ta cũng có thể gán một biến cho một biến khác 

							
							var a = 3; // a mang giá trị 3
							
							a = 'Ocena' // a giờ mang giá trị 'Ocena'

							var b = a; // b có giá trị giống a

							alert(a);
							alert(b);
var a = 3; a = 'Ocena' var b = a; alert(a); alert(b);

Đặt Tên Biến

Trong JS có một quy tắc đặt tên biến là camelCase
Hay là viết thường chữ đầu và viết hoa chữ đầu của những chữ tiếp theo

							
							var userName;
							var user31;


Có 2 ký tự đặc biệt có thể dùng để đặt tên là $ và _ 

							
							var $, _;

JS phân biệt hoa thường trong tên biến
userName và USERNAME là 2 biến hoàn toàn khác nhau.

Các Quy Tắc Đặt Tên Cơ Bản

Nếu là những biến thông thường hoặc là không biết đặt gì thì hãy đặt là camelCase
Nên sử dụng tên biến là tiếng anh và sử dụng cái tên ngắn, dễ hiểu và có ý nghĩa cho giá trị của biến

Đối với những hằng số hay là một biến chỉ tạo 1 lần và không thay đổi giá trị của nó nữa thì người ta có thể viết hoa tất cả.

Đối với những biến chỉ sử dụng nội bộ (không dùng để lấy dự liệu hay lưu dữ liệu từ bên ngoài nhập vào hay từ data lấy xuống) những biến này thường chỉ được dùng để tính toán và có giá trị nội bộ mà thôi
Sẽ đặt tên có dấu _ ở trước 

							
							var firstName = 'Anna'; 
							// userName có thể được lấy từ form 
							// hoặc là từ dữ liệu bên ngoài thì dùng camelCase

							var THEME_DARK = '#000'; 
							// Thường là những giá trị không thay đổi, 
							// được khởi tạo để làm việc thì dùng viết hoa

							var a = 3, b = 4;
							var _sum = a + b; 
							// sum chỉ được dùng để hỗ trợ tính toán và 
							// viết code dùng "nội bộ" thì sẽ đặt với _
							var c = _sum;

Data Types - Kiểu dữ liệu

JS là một ngôn ngữ "dynamic typing" có nghĩa là một biến có thể nhận nhiều kiểu dữ liệu khác nhau
Tại thời điểm này là một string, một thời điểm khác lại là một number....
Chúng ta cũng không cần khai báo kiểu dữ liệu khi khai báo biến.

A number - Kiểu số

Kiểu số là đại diện của cả số nguyên và cả số thực
Có thể thực hiện các phép toán như cộng trừ nhân chia trên số
Ngoài số bình thường thì còn có thêm 3 giá trị đặc biệt khác cũng thuộc kiểu số
Đó là: Infinity, -Infinity, NaN
NaN có nghĩa là Not a Number, là đại diện cho một phép toán lỗi 

							
							alert('abc' * 30);
alert('abc' * 30);

A string - Kiểu chuỗi

String có nghĩa là chuỗi, thường là một đoạn chữ hoặc là một ký tự nào đó
để định nghĩa một string thì chúng ta cần phải bỏ quote
Hiện tại có 3 cách để định nghĩa string
` backtick là một loại mới dùng để lồng thêm biến vào chuỗi dễ dàng hơn 

							
							var a = 'dau nhay don';
							var b = "dau nhay doi";
							var c = `ES6 backticks`;

							// ES6 String Interpolation
							var n1 = 'Memories';
							var n2 = 'Dust';

							alert(`${n1} of ${n2}`);
var n1 = 'Memories'; var n2 = 'Dust'; alert(`${n1} of ${n2}`);

A boolean - Kiểu logic

Kiểu này chỉ có 2 giá trị true hoặc là false
Những phép so sánh cũng sẽ trả về kết quả là kiểu boolean 

							
							var a = 10;
							var b = 20;

							alert(a > b); // false
var a = 10; var b = 20; alert(a > b); // false

null

Null có nghĩa là trống rỗng, không có gì cả, không biết ...
Null khá là đặc biệt vì nó không phải kết quả hay các phép tính của JS tạo thành
Null thường là do lập trình viên viết vào và thường được gán chứ không phải tự nhiên mà có
Dùng null:
Khi một biến có giá trị nào đó, và bằng một lý do nào đó chúng ta không muốn giá trị đó nữa, muốn xóa đi thì sẽ gán nó bằng null hoặc dùng như là một cách cho người khác biết là nó không có giá trị gì cả 

							
							var name = 'Eito'; // khởi tạo name với giá trị 'Eito'

							name = null; // không muốn nữa và xóa nó khỏi bộ nhớ

undefined

undefined là một giá trị có thể sẽ được tạo ra khi viết code trong JS
Khi bạn nhận được giá trị undefined có nghĩa là giá trị đó do code tạo ra, do chính JS nó tạo ra, khác với null
Null là do con người chủ động gán thì nó mới có
Undefined có nghĩa là không xác định, khi bạn tạo một biến ra và không gán giá trị, thì nó sẽ có giá trị undefined, khi bạn truy cập vào một phần tử không có của array hay là object thì bạn cũng sẽ nhận được undefined 

							
							var a; // khai báo không gán giá trị

							var user = {}; // khai báo một object

							alert(a);
							alert(arr[3]); // truy cập vào phần tử không có trong mảng
							alert(user.name); // truy cập vào một thuộc tính không tồn tại
var a; var arr = []; var user = {}; alert(a); alert(arr[3]); alert(user.name);

Về cơ bản chúng ta có thể gán giá trị undefined cho một biến
nhưng đó là điều không được khuyến cáo bởi vì cơ bản chúng ta không việc gì phải gán cho một biến mang giá trị "không xác định"

Type Conversions - Chuyển đổi kiểu dữ liệu

Khi viết chương trình chúng ta có lúc sẽ cần phải chuyển đổi kiểu dữ liệu
Biết cách chuyển đổi kiểu dữ liệu là một phần tất yếu
Kiểu dữ liệu là rất quan trọng, ngoài việc biết cách chuyển đổi còn cần phải biết thêm cả việc theo dõi, và phỏng đoán xem nó đang mang giá trị gì...

Chuyển Sang String

Về cơ bản chỉ cần dùng String(...)
Hầu như tất cả mọi thứ đều có thể chuyển về string
number, array, function .... đều có thể chuyển về string 

							
							var a = [1,2,3,4];
							
							alert(String(a));
var a = [1,2,3,4]; alert(String(a));

Chuyển Sang Number

Như trên, chỉ cần dùng Number()
Những thứ không chuyển đổi thất bại thì trả về NaN 

							
							var a = 'This is the end';
							var b = true;
							var c = '33';

							Number(a) // NaN
							Number(b) // 1
							Number(c) // 33

Chuyển Sang Boolean

Dùng Boolean 

							
							var a = 'This is the end';
							var b = 0;

							Boolean(a) // true
							Boolean(b) // false

Truthy và Falsy
Một giá trị là truthy khi mà chuyển sang Boolean nó là true
Một giá trị là falsy khi mà chuyển sang Boolean nó là false
'' (chuỗi rỗng chứ không phải dấu cách), null, undefined, NaN, 0 --- 5 giá trị này là falsy, còn lại thì là truthy

Đây chỉ là những cách cơ bản để chuyển đổi kiểu dữ liệu
Thực tế khi viết chương trình và sử dụng hàm thì những hàm này cũng sẽ có trả về kiểu dữ liệu khác nhau
Hay là khi cộng một giá trị với một chuỗi thì nó cũng sẽ được chuyển về chuỗi
Nhắc lại một lần nữa việc theo dõi và nắm vững kiểu dữ liệu là hết sức quan trọng trong việc viết code
Mỗi kiểu sẽ có hàm riêng và cũng sẽ hành xử khác nhau

Operator and Comparisons - Phép tính và Phép so sánh

Phép tính và so sánh hoàn toàn cần thiết trong cả cuộc sống và trong lập trình cũng vậy

Các Phép Tính

= --- đây là phép gán
+ --- phép cộng
- --- phép trừ
* --- phép nhân
/ --- phép chia
% --- chia lấy dư
** --- lũy thừa (cẩn thận cái này không hoạt động trên IE11)
++ --- tăng lên 1 đơn vị
-- --- giảm xuống 1 đơn vị
+= --- viết tắt
++ và -- có 2 cách viết là viết trước và viết sau 2 cách viết này cũng sẽ có sự khác nhau

Ngoài thực hiện với số JS còn cho phép thực hiện phép tính với các kiểu dữ liệu khác.
Việc cộng 2 string là trường hợp thường được sử dụng nhất. 

							
							var a = 'Light';
							var b = ' and ';
							var c = 'Dark';

							a + b + c; // 'Light and Dark'
							a + 30; // 'Light30'
							a - 30; // NaN

Các Phép So Sánh

> --- lớn hơn
< --- nhỏ hơn
>= --- lớn hơn hoặc bằng
<= --- nhỏ hơn hoặc bằng
== --- so sánh bằng (cái này thường không để ý đến kiểu)
!= --- ngược với cái ở trên (cái này cũng không để ý đến kiểu dữ liệu)
=== --- so sánh tính cả kiểu dữ liệu ( đúng cả kiểu và giá trị thì sẽ trả về true )
!== --- ngược với cái trên ( khác kiểu hoặc khác giá trị ) ( trừ khi đúng cả kiểu cả giá trị thì mới trả về false )

Với mọi phép so sánh và mọi kiểu dữ liệu thì Boolean ( true hoặc false ) luôn luôn là kết quả, không có lỗi xảy ra
Và chúng ta có thể gán giá trị này cho một biến như bình thường 

							
							var a = 'Try' > 30;

							alert(a) // ????;
var a = 'Try' > 30; alert(a) // ????;

String sẽ được so sánh với nhau dựa vào thứ tự Unicode của nó 

							
							alert('b' > 'a');
alert('b' > 'a');

Higher Order Function

Higher Order Function có nghĩa là function cấp cao.
Đây là những function nhận vào function khác hoặc là function trả về một function khác.
Đây là những hàm rất mạnh và khó sử dụng nhưng nó cũng đóng vai trò quan trọng trong khi làm việc với mảng.

Đối với mảng thì có rất nhiều higher order function được xây dựng sẵn.
Những function này thường sẽ nhận vào một tham số.
tham số đó là một function khác.
Function khác này có thể nhận vào 3 tham số ( thường chỉ dùng một )
tham số 1: chính là cái giá trị hiện tại đang được trỏ tới
tham số 2: chính là số thứ tự của giá trị đó (index)
tham số 3: chính là cái mảng đang sử dụng lệnh
***: tài liệu sẽ viết tắt những hàm này thành arrayCallbackFunc

Function này có thể nhận lên tới 4 tham số (thường chỉ dùng 2)
tham số 1: Accumulator ( giá trị tích lũy từ trả về )
tham số 2: Current Value ( phần tử hiện thời trong mảng )
tham số 3: index ( số thứ tự của phần tử hiện thời )
tham số 4: chính là cái mảng đang chạy lệnh này
Thường gọi là reducer function
Trả về:
Trả về một giá trị duy nhất 

							
							var arr = [1,2,3,4];

							var result = arr.reduce( function(acc, curr) {
								  return acc + curr;
							} )

							// hãy xem cách mà reduce hoạt động
							// đâu tiên là reduce không có initial value
							// reducer function sẽ nhận phần tử đầu tiên trong arr là acc
							// bắt đầu chạy từ phần tử thứ hai

							// vòng 1
							// acc là arr[0] (1)  ---  curr là arr[1] (2)
							// trả về: 3 ( 1 + 2 )
							
							// giá trị trả về sẽ dùng làm acc cho lần tiếp theo
							// vòng 2
							// acc: 3 (trả về) ---  curr là arr[2] (3)
							// trả về: 6 

							// vòng 3
							// acc: 6 (trả về) ---  curr là arr[3] (4)
							// trả về: 10
							
							// kết thúc vì hết phần tử trong mảng
							// kết quả trả về 10
						
							alert(result);

Regular Expressions

Regular Expressions hay gọi tắt là RegExp.
Là những "thần chú" cực kỳ mãnh mẽ khi làm việc với chữ.
Thường được định nghĩa bằng dấu /.../ hoặc tạo dựng với RegExp Object.
RegEx có 2 phần gồm pattern và flag 

						
						var regex = /pattern/; // cái ở trong // là pattern
						var regex = /pattern/gi; // g, i ở ngoài gọi là flag

						// có 2 loại flag thường dùng là
						// i --- bỏ qua hoa thường ( A cũng như a)
						// g --- tìm kiếm toàn bộ ( global ), nếu không có thì chỉ trả về cái đầu tiên tìm được

str.match

Tìm những từ phù hợp trong string, và trả về mảng 

							
							var str = "A love B lOve C LovE D love Z LOVE A";

							// tìm chữ "love" --- có 2 nhưng chỉ trả về 1
							alert(str.match(/love/)); 

							// tìm tất cả chữ love --- có flag g, trả về tất cả
							alert(str.match(/love/g));

							// tìm tất cả chữ love --- có flag g, i bỏ qua hoa thường và trả về tất cả
							alert(str.match(/love/gi));

Character Class

Đại diện cho 1 ký tự duy nhất.

\d
Số từ 0 - 9
\s
Khoảng cách
\w
Thường là chữ latin (a - z)
.
Ký tự nào cũng được
\D
Ngược lại của \d, không phải là số
\W
Ngược lại của \w, không phải là chữ
\S
Ngược lại của \s, không phải là khoảng trắng

Quantifiers

{n}
\d{3} bắt buộc là 3 \d.
{1,4}
\d{1,4} từ một đến 4 \d.
+
Có nghĩa là một hoặc nhiều hơn.
\s+ một hoặc nhiều hơn khoảng trắng.
?
Không có hoặc có một.
*
Không có hoặc có nhiều hơn 0 

							
							var str = '123 asdasf 12344 asdfs';

							alert( str.match(/\d{4,6}/) );
							alert( str.match(/\d+/gi) );
var str = '123 asdasf 12344 asdfs'; alert( str.match(/\d{4,6}/) ); alert( str.match(/\d+/gi) );

/(la)+/: viết như này có nghĩa chữ "la" một hoặc nhiều hơn, lala lalala .... 

							
							var str = 'nonono nono nooooo';

							var str1 = '1a2b3b, 1a2b3bz6ss'
							alert( str1.match(/(\d\w)+/g) );
